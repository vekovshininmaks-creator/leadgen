# Синхронизация данных между машинами и сервером

Кратко, без лишнего риска.

## Что хранится где

- **`tenants/org_<N>/`** — `config.json`, `sent_leads.csv`, `state.json`, сессии Telegram в `sessions/`, логи в `logs/`.
- **SQLite приложения** (путь в окружении / в `web_app.py`) — веб-сессии, подписки, CRM, очереди и т.д.

## Безопасный порядок

1. **Остановите** бота и веб-приложение на обеих сторонах перед копированием SQLite и «живых» CSV — иначе возможно повреждение БД (WAL).
2. Сделайте **бэкап**: вся папка проекта или минимум `tenants/` + файл основной БД.
3. Копируйте **целиком** `tenants/org_<id>/`, чтобы конфиг, журнал лидов, `state` и `sessions` не разъехались.
4. **Не сливайте** две разные SQLite «как есть», если в обеих уже есть данные — конфликт первичных ключей. Для одной организации выберите одну БД источником или переносите выборочно по таблицам с фильтром `org_id`.
5. Один **Telegram-аккаунт**: не запускайте параллельно две копии с одинаковыми `sessions/*` из разных хостов — риск flood/бана и рассинхрона `state.json` / `last_seen_msg_id`.
6. После обновления кода при первой записи бот может **добавить колонки** в `sent_leads.csv` (автомиграция).

## Окно браузера и мониторинг
