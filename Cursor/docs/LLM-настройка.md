# LLM: кто настраивает и как пользоваться

Данные для запросов к нейросети (текст промптов, фрагменты переписок) обрабатываются на стороне выбранного вами API-провайдера. Перед настройкой уточните в документации провайдера условия хранения и обработки данных в **РФ** (в т.ч. **152-ФЗ**), если для вас это критично. Ключ API хранится в файле настроек организации на **вашем** сервере, не передаётся в интерфейс ролям без прав администратора.

## Администратор организации

1. Откройте **«Контакты» → «Настройки и лимиты»** и переключитесь на вкладку **«Подключения (Telegram + LLM)»** (тот же конфиг, что и у всей org).
2. В блоке **LLM** включите переключатель. При желании выберите **провайдера (пресет)** — подставятся Base URL и модель.
3. Укажите **API key**, при необходимости скорректируйте **Base URL** и **Модель** вручную.
4. Нажмите **«Сохранить настройки»** (основная кнопка сохранения на странице настроек).

Ключ хранится в конфиге организации на сервере. Менеджеры и клиенты **не видят** ключ и **не могут** его менять через интерфейс.

## Менеджер

- Системный админ назначает менеджера **в org клиента** (Admin → Пользователи), после чего менеджер работает в данных этой org.
- На странице **«Поиск каналов»** можно **сгенерировать ключевые слова по нише**, если администратор организации уже настроил LLM.
- Если генерация недоступна — попросите администратора организации настроить LLM на вкладке «Подключения».

## Клиент

- Обычно достаточно работы с ботом и настройками, доступными по роли.
- Генерация фраз для поиска каналов — у **менеджера** и **администратора** (страница «Поиск каналов»).

## Где взять API key (пример: OpenAI)

1. Зайдите на [platform.openai.com](https://platform.openai.com/).
2. Подключите способ оплаты при необходимости.
3. Раздел **API keys** → создайте ключ.
4. Вставьте ключ в поле **API key** на вкладке «Подключения» и сохраните настройки.

Другие провайдеры с совместимым эндпоинтом `/v1/chat/completions` можно указать через **Base URL** и **Модель** по их документации.

**Cerebras** (`https://api.cerebras.ai/v1`): актуальные id моделей см. [Supported Models](https://inference-docs.cerebras.ai/models/overview) (например `gpt-oss-120b`, `llama3.1-8b`). Старое имя `llama-3.3-70b` в публичном API может быть недоступно — при 404 смените модель в **«Настройки и лимиты» → «Подключения»**.

**Groq** (`https://api.groq.com/openai/v1`): для Llama 3.3 70B укажите `llama-3.3-70b-versatile`, а не короткое `llama-3.3-70b`.
