# Запуск Leadgen (Windows / локально)

## 1. Виртуальное окружение Python

В PowerShell перейдите в папку проекта (где лежат `web_app.py`, `requirements.txt`), затем:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
```

Если политика запрещает скрипты:

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

## 2. Зависимости

```powershell
pip install -r requirements.txt
```

### Переменные окружения (безопасность, волна C)

- `FLASK_SECRET_KEY` — обязателен в **prod**; для CSRF, подписи второго шага 2FA и сессий.
- `ENV=prod` — включает **Secure** у cookie `auth_token` (только по HTTPS / TLS).
- `HTTPS=1` — то же, если нельзя задать `ENV` (например за reverse-proxy).

## 3. Веб-консоль (основной способ)

```powershell
python web_app.py
```

Откройте в браузере: **http://127.0.0.1:5173**

Запускайте команду **из папки проекта**, где лежат `web_app.py` и `templates` (у вас это обычно каталог с Яндекс.Диском, а не произвольная копия).

### Процессы (почему «вижу только один»)

- **Один процесс `python.exe`** — это нормально: это только веб-сервер Flask.
- **Второй процесс** появится **после «Старт»** на странице «Управление»: тогда поднимается дочерний процесс с `telegram_leadgen_bot.py`. Пока бот не запущён — в диспетчере задач будет один Python от консоли.

Если после обновления кода страница «как раньше», сделайте в браузере **жёсткое обновление**: `Ctrl+Shift+R` или `Ctrl+F5`.

### Почему панель открывается, когда «всё выключено»

- **Веб-панель** (`web_app.py`) и **Telegram-бот** — разные процессы. Кнопка «Стоп» в интерфейсе гасит только бота, **не** веб-сервер.
- Закрытие вкладки или встроенного браузера в Cursor **не останавливает** сервер: он продолжает работать, пока жив процесс `python` с Flask.
- Если когда-то запускали `python web_app.py` в терминале и окно просто свернули, процесс может **всё ещё слушать порт 5173** — тогда любой браузер (в том числе внутри Cursor) по-прежнему покажет сайт.

Чтобы панель **реально** перестала открываться, нужно завершить процесс веб-сервера (см. раздел «Остановка» ниже).

- Войдите или зарегистрируйтесь.
- Для **каждой организации** в **Настройках** укажите свои `api_id`, `api_hash` и телефон (у новых org поля по умолчанию пустые — это нормально для приватности).
- Файлы сессии Telegram (`.session` и журнал) хранятся в каталоге **`sessions/`** в корне проекта (по одному базовому имени на организацию, например `sessions/leadgen_org_1_session`). Вручную в конфиге указывать путь к сессии не обязательно: веб-панель передаёт его боту сама.
- Бота можно запускать со страницы **Управление**.

### Организация пользователя (менеджер, тестировщик и др.)

- В **Admin → Пользователи** задайте **организацию и роль в org** (блок «Организация и роль» → **«Сохранить роль в org»**) — это **`POST /api/admin/membership/set`**; иначе пользователь может остаться в Default org.
- Для платформенного **manager** в таблице `users` по желанию используйте узкий сценарий **«Назначить в эту org»** (`POST /api/admin/manager/assign-org`) — то же назначение membership **manager**, если удобнее старый поток.
- В сайдбаре у менеджера может быть подсказка, пока не назначена org клиента. В API `GET /api/auth/me` есть поля `org_name` и `manager_on_default_org`.

## 4. Остановка веб-панели

1. **Если помните терминал**, где запускали `python web_app.py`: переключитесь в него и нажмите **`Ctrl+C`**.
2. Если терминал закрыт, а сайт всё ещё открывается — найдите процесс на порту **5173** и завершите его (PowerShell):

```powershell
$p = Get-NetTCPConnection -LocalPort 5173 -ErrorAction SilentlyContinue | Select-Object -ExpandProperty OwningProcess -Unique
if ($p) { Stop-Process -Id $p -Force }
```

Проверка, кто слушает порт:

```powershell
Get-NetTCPConnection -LocalPort 5173 | Select-Object OwningProcess
Get-Process -Id (Get-NetTCPConnection -LocalPort 5173).OwningProcess
```

После этого `http://127.0.0.1:5173` перестанет отвечать, пока снова не запустите `python web_app.py`.

## 5. Запуск только движка бота (для отладки)

Обычная работа — через веб-страницу «Управление». Если нужен ручной запуск, укажите пути к файлам **вашей** организации, например `tenants\org_2\`:

```powershell
python telegram_leadgen_bot.py --run-now --force-unlock --config-path tenants\org_2\config.json --state-path tenants\org_2\state.json --csv-path tenants\org_2\sent_leads.csv --log-path tenants\org_2\logs\bot.log --session-name sessions\leadgen_org_2_session
```

`--session-name` — это **базовый путь без расширения** `.session` (как в Telethon); файлы появятся как `sessions\leadgen_org_2_session.session` рядом с проектом.

В конфиге должны быть заполнены Telegram API, телефон и список чатов.

## 6. Порт занят (5173)

Закройте старый процесс или задайте другой порт:

```powershell
$env:PORT=8080; python web_app.py
```
