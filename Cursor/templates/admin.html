Перезапуск сайта

Только для платформенного администратора. На несколько секунд сайт станет недоступен — сессии и кэш не теряются (сохраняются в БД). Метод перезапуска задаётся переменными окружения на сервере, см. подсказку ниже.

Как настроить на сервере
  • systemd (рекомендуется): LEADGEN_RESTART_COMMAND="systemctl restart leadgen.service" и право на эту команду у юзера сервиса (через sudoers или своя unit-зависимость).
  • pm2 / supervisor: LEADGEN_RESTART_COMMAND="pm2 restart leadgen" или supervisorctl restart leadgen.
  • gunicorn --reload-extra-file FLAG: LEADGEN_RESTART_TOUCH_FILE="/run/leadgen/reload.flag" — приложение «трогает» файл, gunicorn делает graceful reload.
  • gunicorn master (SIGHUP): LEADGEN_RESTART_PARENT_SIGNAL="HUP" — отправляем сигнал родителю.
  • Одиночный процесс: LEADGEN_ALLOW_SELF_EXEC=1 — текущий процесс перезапускает сам себя (только если он один и не воркер gunicorn).

Admin Dashboard

Сводка по всей системе: пользователи, организации, подписки и активность по CSV. У каждого пользователя можно задать организацию и роль в org (client / manager / tester / admin орг) через блок «Организация и роль» и API POST /api/admin/membership/set. Отдельно роль manager в таблице users — платформенная; менеджеров клиента по-прежнему можно назначать так же.

Статусы подписок

Сколько организаций в каждом статусе подписки (active, trial, paused, expired, banned).

Активность по организациям (CSV)

Строки в sent_leads.csv по каждой org — приблизительная метрика записанных событий/лидов (зависит от формата CSV).

Что означают метрики

Тарифы (планы в БД)

Лимиты из таблицы plans применяются ко всем организациям с этим plan_id. Сохранение — сразу в базу (CSRF по сессии).

Пользователи

Тариф и статус подписки задаются по организации пользователя (кнопка «Применить тариф» в строке). Фильтр по дате — локальное время браузера; created_at из базы.
