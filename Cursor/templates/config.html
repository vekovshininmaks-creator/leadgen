Настройки и лимиты

Тариф и что реально применится

Лимиты в форме ниже при сохранении ограничиваются тарифом (чаты, личные сообщения в сутки/месяц, мин. интервал). Ниже — потолки плана и текущие значения в файле конфигурации.

Тарифные лимиты применяются при сохранении и при запуске бота.

Пресеты настроек

Готовые наборы лимитов, шаблонов, ключевых слов, расписания, фильтров поиска каналов и промптов LLM. Применение делает deep-merge: индивидуальные поля (Telegram-аккаунты, API-ключ LLM, целевые чаты) не трогаются. Перед применением показываем превью изменений.

Безопасный режим и подпись

Тестовый запуск Согласование перед отправкой Какие этапы через согласование

Тестовый запуск — бот ищет лиды и пишет в журнал, но не инициирует реальные личные переписки. Подпись — подставляется в шаблоны (этап 3) и в подсказки LLM. Согласование — для отмеченных этапов текст сначала в «Офферы»; снятая галочка этапа = отправка сразу в очередь бота без подтверждения. Мастер-тумблер выше должен быть включён.

Поиск каналов: качество выдачи

Только для администратора организации. Включение режима добавляет запросы к чатам обсуждений (или к мегагруппам) при глобальном поиске — медленнее, но отсекает «мёртвые» и одноавторские витрины. Сохраните настройки и выполняйте поиск при остановленном боте.

Telegram API (только эта организация)

api_id и api_hash с my.telegram.org. У новой организации при первом открытии настроек поля пустые — корневой config не подставляет чужие ключи.

Несколько аккаунтов

У каждого аккаунта — отдельный логин Telethon и файл в sessions/. Активный — тот, с которым работает мониторинг (запущенный бот). В «Поиск каналов» можно отметить несколько и объединить выдачу.

При смене «Активный (бот)» форма переключается на этот же аккаунт (черновик предыдущего аккаунта сохраняется в буфере). Чтобы править другой аккаунт без смены активного — используйте только «Редактировать».

Подставит Base URL и модель; ключ вводится отдельно.
Промпты для генерации (system / user)

Редактирует только администратор организации. В шаблонах «user» можно использовать плейсхолдеры в фигурных скобках ({partner} и т.д.) — их подставляет система; неизвестное имя даст пустую строку. Поле, оставленное как при загрузке страницы, совпадает с текстом по умолчанию и в файл конфигурации не дублируется.
